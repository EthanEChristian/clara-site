Retaking rules for developers

Rules can greatly simplify business logic, but often come in the form of limited mini-languages and complex tools. Let's take them back.

Simple

Clara brings the best parts of rule engines into your code without the weight of an enterprise system.

Expressive

Clara keeps simple rules simple, but users can write rich logic or invoke external libraries when they need to.

It's just Clojure

Rules are just Clojure code, and can be managed and reused like anything else.

{% highlight clojure %} (defrule work-approval "Check order approval." [WorkOrder (= type :repair) (= ?order-id order-id)] [:not [ApprovalForm (= ?order-id approved-id) (= formname "27B-6")]] => (insert! (->MissingApproval ?id "No 27B-6."))) {% endhighlight %} Check out the developer guide.

Easily used from Java

Just include your rules as a resource in your and use Java Beans or Clojure records as facts.

{% highlight java %} List results = RuleLoader.loadRules("clara.examples") .insert(facts) .fireRules() .query("clara.examples/approvals"); for (QueryResult result: results) System.out.println(result.getResult("?id")); {% endhighlight %} See the Java documentation for more.

And many other features...

Clara supports the features you'd expect from a rule engine, such as support for truth maintenance, durability, rule activation explanations, accumulators to reason across sets of facts, and others. It also brings in the best ideas from functional programming, with strong immutability support making rule sessions easy to compose and work with.
